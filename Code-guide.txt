### Code-guide for MultiKEY - EN ###

#Branching - One Guy To Merge
One person merges the code, kind of like a code review
Use branching

#Language
Should always be English

#Comments
Comments are written in Danish, maximum 1 sentence.
Headlines for sections are in caps lock.

#Syntax
function() {
// Some code
};

#Space between code
There should always be 1 "ENTER" between functions

#Variables
Descriptive names of variables

#Camel case
You write variable and functions using camel case. Meaning the first word starts with a small letter, and the subsequent words start with a big
ex. tinyGoodVariable

#NO var PLEASE!

#Comment first
First write the comment for what the code will do, and then you can start coding it

### Kode-guide for MultiKEY - DA ###

#Sprog
Skal altid være engelsk

#Kommentarer
Kommentarer er skrevet på dansk, og fylder maks. 1 sætning.
Overskrifter til sektioner er i caps lock.

#Syntax
function() {
// Some code
};

#Afstand mellem kode
Der skal altid være 1 "ENTER" mellem funktioner

#Variabler
Beskrivende navne for variabler

#Camel case
Man skriver variabler og funktioner med camel case. Dette betyder at det første ord altid starter med småt, og de resterende med stort.
Eks. tinyGoodVariable

#INGEN var PLEASE!

#Kommentar først
Først skriver du kommentaren til hvad koden gør, bagefter koder du det så.
